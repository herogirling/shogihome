# USI Engine Bridge Server セットアップ

## 1. 準備

1. Python 3.10 以上をインストール
1. ファイアウォールで使うポートを受信許可する
1. ルーターでポート開放をする

このプログラムは、外部からローカル環境の将棋エンジンにアクセスする機能を提供します。
ポート開放できない場合は、例えば外出中など外部のネットワークから使えません。

## 2. サーバーを起動する

サーバーにしたいPCから`scripts/usi_engine_bridge_server.py`を実行します。

```bash
python scripts/usi_engine_bridge_server.py --engine "scripts/usi-engine.exe"
```

引数:

- `--engine`: USIエンジン実行ファイルのパス（必須）
- `--host`: 待受アドレス（既定: `0.0.0.0`）
- `--port`: 待受ポート（既定: `22391`）

## 3. ShogiHome 側の設定

エンジン管理ダイアログの「追加」で、サーバーアドレスを入れます。

- 同じPCで使う: `127.0.0.1:22391`
- LAN内の別端末から使う: `<サーバーPCのIP>:22391`
  例: `192.168.1.10:22391`
- ポート開放している場合はLAN外からも使用可能です。

## 4. 動作確認

起動後に次のログが出ればOKです。

```text
[usi-engine-bridge-server] listening on http://0.0.0.0:22391
```

そのあと、ShogiHome側でエンジン追加が成功するか確認します。

## 5. 停止

起動したターミナルで `Ctrl+C` を押します。

## 6. うまくいかないとき

1. サーバーが起動中か確認する
1. 入力したアドレスとポートが合っているか確認する（`22391`）
1. ファイアウォールとルーターでそのポートが許可されているか確認する
